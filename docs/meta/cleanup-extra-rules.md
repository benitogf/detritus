# Cleanup Extra Rules & Hooks

Remove everything generated by `setup-extra-rules`:

- Rule files: `~/.claude/rules/detritus-*.md`
- Hook scripts: `~/.claude/hooks/detritus-*.sh` and `~/.claude/hooks/detritus-*.ps1`
- Hook entries in `~/.claude/settings.json` that reference `detritus-*` scripts

Does NOT touch:

- Any file or hook entry that doesn't carry the `detritus-` prefix
- Baseline settings managed by `setup-superpowers` (`permissions.deny`, `statusLine`, `effortLevel`, `alwaysThinkingEnabled`, `showThinkingSummaries`, `autoMode`)

## Phase 1: Inventory

Build a list of what exists and would be removed. Run silently:

1. List files matching `~/.claude/rules/detritus-*.md`
2. List files matching `~/.claude/hooks/detritus-*` (any extension)
3. Read `~/.claude/settings.json` (parse as JSON; if invalid/missing, skip the settings step entirely — there's nothing to clean)
4. Walk hook entries; collect any whose `command` references `~/.claude/hooks/detritus-*`

## Phase 2: Confirm

Show the user a single message listing everything that will be removed:

```
The following will be removed:

Rule files (N):
  - ~/.claude/rules/detritus-go-style.md
  - ~/.claude/rules/detritus-workflow.md
  ...

Hook scripts (N):
  - ~/.claude/hooks/detritus-gofmt.sh
  ...

settings.json hook entries (N):
  - PostToolUse matcher "Edit|Write" → bash ~/.claude/hooks/detritus-gofmt.sh
  ...

Baseline settings (deny list, statusLine, effortLevel, autoMode) will NOT be touched.

Proceed? (yes/no)
```

If the inventory is empty in all three categories, say so and exit without prompting — no work to do.

If the user says no, exit without changes.

## Phase 3: Remove

Only after explicit confirmation:

1. Delete each rule file (use the dedicated file-delete tool, not `rm`).
2. Delete each hook script.
3. Update settings.json:
   - Re-read the file (the user could have edited between Phase 1 and now).
   - Walk every hook array, remove any entry whose `command` references `~/.claude/hooks/detritus-*`.
   - If a hook array becomes empty, remove the parent matcher object too.
   - Never reorder, deduplicate, or rewrite non-detritus entries.
   - Write back with proper JSON formatting.

## Phase 4: Report

Concise summary:

1. **Removed** — count of rule files, hook scripts, and settings entries deleted
2. **Preserved** — confirm baseline settings are intact
3. **Next step** — mention that `setup-extra-rules` can regenerate everything if needed

## Important Constraints

- Skill is **destructive** — always confirm before deleting (Phase 2). The only exception is when there's nothing to remove.
- NEVER delete files that don't start with `detritus-`
- NEVER touch `permissions`, `statusLine`, `effortLevel`, `autoMode`, or any other settings that aren't hook entries
- NEVER delete the entire `hooks` block in settings.json — only remove `detritus-*`-referencing entries
- Skill is **re-runnable** — running it on an already-clean system should be a no-op
